# Mistake Notes

## 2026-03-30

### 1. Wrong test case naming
- Initially created `home-test5-cp-artifacts-v2` instead of following the existing naming convention
- **Lesson**: V2 cases should be named `test10-cp-artifacts-v2`, `test13-cp-artifacts-v2` etc. (without "home-" prefix, using sequential test numbers)

### 2. Used wrong image in copy pod
- Set `image: alpine:latest` in `expected-copy-pod.yaml` for non-home test cases
- **Lesson**: Only home cases use `alpine:latest` for CP_cp_image. Non-home cases should use `swr.cn-southwest-2.myhuaweicloud.com/modelfoundry/alpine:3.23.3`

### 3. Added `CP_cp_image` to non-home case
- Added `export CP_cp_image="alpine:latest"` to env.sh of a non-home test case
- **Lesson**: `CP_cp_image` is only used for home-test* cases, not regular test cases

### 4. Wrong CP_runs_on format
- Used `CP_runs_on="amd64-cpu-1-mem-512M"` (home format) instead of `CP_runs_on="amd64"` (regular format)
- **Lesson**: Home cases use full format like `amd64-cpu-1-mem-512M`, regular cases just use `amd64`

### 5. Expected files didn't match generated output
- Expected workflow YAML had wrong image, resources, and PVC names compared to what the code actually generates
- **Lesson**: When creating new test cases, always run the Go test first to see what the code actually generates, then update expected files to match

### 6. Wrong PVC name in eval.sh
- Hardcoded wrong PVC name `pipeline-artifact-temp-home13-artifacts-123-68bd8d0918579208` instead of actual generated `pipeline-artifact-temp-home13-artifacts-123-0294a695be5994ea`
- **Lesson**: Always use the actual hash suffix that the code generates

### 7. Wrong resource validation in eval.sh
- Checked for `cpu: "1"` and `memory: 512Mi` when the v2 template generates `cpu: "8"` and `memory: 8Gi`
- **Lesson**: V2 template uses different resource defaults than home template

### 8. Removed wrong test case
- Removed `test13-cp-artifacts-v2` from test-cases.json when asked to "remove home case"
- **Lesson**: Clarify which cases to remove before modifying test-cases.json

### 9. Dirty $ in clean.sh
- Line 16 had `artifact_dest="${WORKSPACE}$"` causing path to have trailing `$`
- **Lesson**: Always verify variable expansion is correct in shell scripts

### 10. get_workflows_using_pvc() bug
- JQ query didn't check nested volumeMounts in templates (script.volumeMounts)
- **Fix**: Updated to check:
  - `.spec.volumes[]?.persistentVolumeClaim.claimName`
  - `.spec.templates[]?.script.volumeMounts[]?.persistentVolumeClaim.claimName`
  - `.spec.templates[]?.container.volumeMounts[]?.persistentVolumeClaim.claimName`
