# Namespace Mapping Design

## Summary

Add namespace mapping in the submitter to route workflows to different namespaces based on repository name. The converter does NOT change - it already sets the `jobRepositoryName` label from `CP_repo_url`. The submitter reads this label and maps to the appropriate namespace.

## Motivation

Different repositories need to run in different namespaces for isolation and resource management. The RAGSDK repo and test repos should go to `ragsdk` namespace, while other repos default to `argo`.

## Architecture

### Data Flow

```
CP_repo_url → Converter → jobRepositoryName label → workflow.yaml
workflow.yaml → Submitter → read label → map to namespace → submit to namespace
```

### Components

1. **Converter** (no changes needed):
   - Already extracts repo name from `CP_repo_url` via `extractOrgRepoVolcano()`
   - Already sets `jobRepositoryName` label in Volcano Job metadata
   - Does NOT set namespace in Job or Secret manifest

2. **Submitter** (changes required):
   - Read `jobRepositoryName` label from workflow.yaml before submit
   - Apply namespace mapping logic
   - Submit Job and apply Secret to mapped namespace

3. **Namespace Mapping** (hardcoded in submitter):
   - Repo contains "ragsdk" → namespace "ragsdk"
   - Repo starts with "test" → namespace "ragsdk"
   - Repo == "ascend-ragsdk" → namespace "ragsdk"
   - Default → namespace "argo"

## Implementation Details

### Submitter Changes

#### 1. Add namespace mapping function

```go
func getNamespaceFromRepoName(repoName string) string {
    repoNameLower := strings.ToLower(repoName)
    
    if strings.Contains(repoNameLower, "ragsdk") {
        return "ragsdk"
    }
    if strings.HasPrefix(repoNameLower, "test") {
        return "ragsdk"
    }
    
    return defaultNamespace // "argo"
}
```

#### 2. Extract repo name from workflow.yaml

Add function to read `jobRepositoryName` label:

```go
func getRepoNameFromWorkflow(workflowPath string) (string, error) {
    data, err := os.ReadFile(workflowPath)
    if err != nil {
        return "", err
    }
    
    var job struct {
        Metadata struct {
            Labels map[string]string `yaml:"labels"`
        } `yaml:"metadata"`
    }
    
    if err := yaml.Unmarshal(data, &job); err != nil {
        return "", err
    }
    
    return job.Metadata.Labels["jobRepositoryName"], nil
}
```

#### 3. Update `run()` function

Before submitting, determine namespace:

```go
repoName, err := getRepoNameFromWorkflow(cfg.WorkflowOutput)
if err != nil {
    // Log warning, use default namespace
    fmt.Fprintf(os.Stderr, "WARN: Could not read repo name from workflow: %v\n", err)
} else if repoName != "" {
    mappedNamespace := getNamespaceFromRepoName(repoName)
    if mappedNamespace != cfg.Namespace {
        fmt.Printf("Mapping repo '%s' to namespace '%s'\n", repoName, mappedNamespace)
        cfg.Namespace = mappedNamespace
    }
}
```

#### 4. Add YAML parsing dependency

Add `"go.yaml.in/yaml/v3"` import to submitter if not already present.

### Test Updates

1. Add unit tests for `getNamespaceFromRepoName()`:
   - Test "ascend-ragsdk" → "ragsdk"
   - Test "test1", "test2", "test-repo" → "ragsdk"
   - Test "random-repo" → "argo"
   - Test empty string → "argo"

2. Add unit tests for `getRepoNameFromWorkflow()`:
   - Test workflow with label
   - Test workflow without label
   - Test invalid YAML

### Secret Manifest Changes

The secret manifest generated by converter uses empty namespace (defaults to "argo" in `BuildSecretManifest`). The submitter applies the secret with `-n <mappedNamespace>`, so the namespace in manifest doesn't matter - kubectl apply uses the flag namespace.

No changes needed to secret_manager.go.

## Edge Cases

1. **Missing label**: If workflow.yaml has no `jobRepositoryName` label, use default namespace `argo`
2. **Empty repo name**: If label exists but is empty string, use default namespace `argo`
3. **Unknown repo**: If repo name doesn't match any pattern, use default namespace `argo`

## Testing Strategy

1. Unit tests for namespace mapping function
2. Unit tests for workflow YAML parsing
3. Integration test: Create workflow with specific repo label, verify submit uses correct namespace
